# military_training_bot

Краткое техническое описание проекта Telegram-бота для сопровождения учений.

## Назначение проекта

Проект автоматизирует выставление, хранение, корректировку, агрегацию и просмотр баллов студентов.

Система должна обеспечивать:

- регистрацию пользователей в боте;
- разграничение ролей;
- безопасный доступ преподавателей;
- выставление баллов в нескольких режимах;
- детализацию баллов старшим студентом;
- просмотр результатов;
- напоминания о невнесённых данных;
- сводную агрегацию результатов;
- экспорт отчётов.

---

## Роли в системе

### Администратор

Отвечает за настройку учений и управление доступом.

Должен уметь:

- создавать новый сезон учений;
- добавлять учебные точки, группы, преподавателей, студентов;
- назначать старших студентов;
- разграничивать роли;
- просматривать общие результаты;
- экспортировать данные;
- сбрасывать или архивировать прошлые данные.

### Преподаватель

Работает только со своей группой или учебной точкой.

Должен уметь:

- создавать оценочные события;
- выставлять баллы каждому студенту;
- выставлять общий средний балл;
- выбирать способ распределения среднего балла;
- корректировать уже внесённые баллы;
- смотреть текущие результаты своей группы.

### Студент

Обычный участник учений.

Должен уметь:

- смотреть свои результаты;
- смотреть историю начислений;
- смотреть текущий итоговый балл.

### Старший студент

Студент с дополнительными полномочиями.

Должен уметь:

- распределять средний балл внутри своей подгруппы;
- отправлять распределение на подтверждение;
- смотреть статус распределения.

---

## Основная логика работы проекта

1. Администратор создаёт новый сезон учений.
2. Назначаются учебные точки, преподаватели, группы и студенты.
3. Все пользователи запускают бота и проходят привязку к системе.
4. Преподаватель создаёт оценочное событие.
5. Баллы могут вноситься в одном из трёх режимов:
   - индивидуально каждому студенту;
   - средний балл, одинаковый для всех;
   - средний балл с распределением старшим.
6. Система сохраняет все изменения и ведёт журнал действий.
7. Итоговые баллы автоматически агрегируются.
8. Пользователи получают напоминания и могут просматривать результаты.
9. Администратор формирует итоговые отчёты.

---

## Режимы выставления баллов

### 1. Индивидуальный режим

Преподаватель указывает балл отдельно каждому студенту.

### 2. Средний балл поровну

Преподаватель задаёт одно значение, которое автоматически применяется ко всем студентам группы.

### 3. Средний балл с делегированием

Преподаватель задаёт общий балл, после чего старший студент распределяет его между участниками группы.

## Требования к проекту

Проект должен быть:

- структурированным;
- расширяемым;

## Используемый стек

- **aiogram** — Telegram-бот;
- **FastAPI** — HTTP API, webhook и служебные маршруты;
- **SQLite** — база данных;
- **SQLAlchemy** — работа с БД;
- **pandas** — агрегация и формирование отчётов;
- **Telethon** — опционально для дополнительных Telegram-сценариев;
- **APScheduler** — напоминания;
- **Alembic** — миграции базы данных.

## Папка `bot/`

Содержит всё, что связано с Telegram-ботом.

```text
bot/
├── factory.py
├── middlewares/
├── filters/
├── handlers/
├── keyboards/
├── states/
└── services/
```

### `factory.py`

Фабрика бота.

Должен:

- создавать экземпляр `Bot`;
- создавать `Dispatcher`;
- подключать роутеры;
- подключать middleware;
- возвращать готовый объект бота.

### `bot/middlewares/`

Промежуточные обработчики Telegram-запросов.

- `auth.py` — проверка, зарегистрирован ли пользователь;
- `role_guard.py` — проверка, имеет ли пользователь право на действие;
- `throttling.py` — ограничение частоты запросов, защита от спама.

### `bot/filters/`

Пользовательские фильтры aiogram.

- `role.py` — фильтрация обработчиков по роли;
- `state_filters.py` — фильтры по состояниям FSM.

### `bot/handlers/`

Основные сценарии взаимодействия с ботом.

- `common.py` — `/start`, `/help`, `/me`;
- `admin.py` — команды администратора;
- `teacher.py` — команды преподавателя;
- `student.py` — команды студента;
- `leader.py` — команды старшего студента;
- `reports.py` — формирование и отправка отчётов.

### `bot/keyboards/`

Клавиатуры Telegram.

- `common.py` — общие кнопки и меню;
- `admin.py` — кнопки для администратора;
- `teacher.py` — кнопки для преподавателя;
- `student.py` — кнопки для студентов и старших.

### `bot/states/`

FSM-сценарии.

- `auth.py` — состояния регистрации и входа;
- `teacher_score_flow.py` — пошаговое внесение баллов преподавателем;
- `leader_distribution_flow.py` — пошаговое распределение баллов старшим студентом.

### `bot/services/`

Сервисные функции бота.

- `notifier.py` — отправка напоминаний и уведомлений;
- `bot_sender.py` — унифицированная отправка сообщений, файлов и служебных уведомлений.

## Что проект должен делать в целом

1. регистрировать пользователей;
2. разграничивать роли;
3. защищать доступ преподавателей паролем;
4. позволять преподавателям вносить баллы;
5. позволять старшим студентам распределять делегированные баллы;
6. хранить историю изменений;
7. автоматически пересчитывать итоговые результаты;
8. показывать результаты пользователям согласно их роли;
9. отправлять напоминания;
10. формировать сводные отчёты для администратора.

В проекте обязательно должны быть:

- чёткое разделение ролей;
- три режима выставления баллов;
- возможность корректировки данных;
- безопасная авторизация преподавателя;
- хранение данных в БД;
- логика агрегации результатов;
- напоминания;
- экспорт результатов;
- понятная иерархия файлов;
- разделение Telegram-логики, бизнес-логики и работы с БД.
